# 07 读稿会、Pitch 与前采

## 核心意思

创意协作需要高效会议、清晰权责，以及把想法卖出去的能力。好的内容人不只是有想法，还能让别人理解、信任并使用这个想法。

## 关键框架

### 读稿会规则

适用于脚本会、选题会、创意会、内容复盘会。

1. 带东西来。不要空手开会。
2. 目标明确：这次会要把作品推进到哪里。
3. 谁负责，听谁的。
4. 用更好的想法否定别人，而不是只说“不行”。
5. 尊重时间。如果出去改稿更有效，就出去改。

### 创作需要“独裁”

创作可以集思广益，但最终责任不能扩散。

- 群体讨论负责增加选项。
- 负责人负责做决定。
- 上台或发布的人最终对自己的表达负责。

### Pitch 是把有用的想法卖出去

Pitch 不是骗别人，而是用专业判断做出真正对对方有帮助的方案，再帮助对方理解它为什么有用。

它适用于领导、同事、客户、嘉宾、平台、赞助商，也适用于作品发布前的内部说服。

### 前采 = pitch + 创作

前采既要建立信任，也要生成可用素材。

- 提前准备。
- 给选择，不给单一的是/否问题。
- 多问事实和行为，少问抽象原因。
- 在现场捕捉可用表达。
- 趁对方在场确认能不能用。

## 关键概念

- **读稿会**：把稿子读出来、判断、修改和推进的会议。
- **谁负责听谁的**：决策权跟着责任走。
- **是的，而且**：不是不能否定，而是要用更好的方向接上。
- **pitch**：让一个想法进入别人的脑子和行动。
- **前采**：事前采访，既建立信任，也生产素材。
- **有选择的真实**：说真话，但不是所有真话都必须说。

## 心智模型

- 用 **带稿子来，否则不开会** 防止空泛头脑风暴。
- 用 **更好想法打败“不行”** 训练反馈质量。
- 用 **给选择** 说服别人：让对方在准备好的方案里选。
- 用 **问发生了什么** 替代过早问“为什么”。
- 用 **爱上角色** 对待嘉宾、客户或受访者：他们是帮你完成作品的人。

## 反模式

- **空手开会**：没人先做出初稿，只能空聊。
- **没有负责人**：大家都参与，但没人拍板。
- **模糊否定**：只说不行，不给方向。
- **只问观点**：过早问抽象问题，让受访者戒备。
- **公开吐槽合作者**：破坏未来信任和职业信用。

## 可执行结论

1. 创意会议要围绕具体稿子或具体决策。
2. 负责人必须有最终决定权。
3. Pitch 是创意工作的一部分，不是脏活。
4. 好前采同时生产信任和可用素材。
5. 合作后的专业态度会保护未来机会。
